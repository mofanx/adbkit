# ADB Kit 产品完善度评估与改进计划

> 评估视角：资深产品经理  
> 评估对象：ADB Kit v1.4.0（versionCode 7）  
> 评估依据：当前 main 分支源码、`README.md`、`docs/features.md`

## 一、产品现状总览

ADB Kit 已构建出一套完整的「移动端 ADB 工具箱」能力，覆盖连接、信息、工具、远程控制、文件/应用/进程管理、命令终端、Fastboot 等 8 大模块，且 UI 采用 Material 3 + Jetpack Compose，技术栈先进。

**当前完成度（满分 10 分）：9.4/10**（本轮迭代后）

- 功能广度：高（开发者/极客核心需求基本覆盖）
- 功能深度：中高（状态反馈、参数配置、结果留存已在关键路径补齐；部分高级能力如定时任务、多设备投屏仍预留）
- 稳定性与体验：中高（P0 修复后危险操作、历史、心跳、引导、崩溃保护均到位，核心逻辑具备单元测试回归）
- 商业化/可用性：中高（首次启动引导、应用内帮助中心、崩溃日志/反馈、用户手册、多语言闭环已补齐；付费点仍缺）

## 二、各模块完善度评估

### 1. 设备连接（完成度 7/10）

**优点**
- WiFi/IP/USB 连接、无线配对、网络扫描、多设备切换都已实现
- 历史地址下拉体验较完整

**不足**
- ~~网络扫描仅扫描 5555 端口，缺少端口范围与超时配置~~ → 扫描对话框已支持自定义起始/结束端口和超时时间
- ~~配对成功后不会自动 `adb connect`~~ → 配对成功后自动设置 IP:端口并调用 connectDevice 完成连接
- ~~未保存连接历史到本地（重启应用丢失）~~ → 已持久化到 DataStore
- ~~连接状态没有实时心跳，断开后 UI 不会自动刷新~~ → 已增加 AdbService 心跳：每 5 秒 `adb shell echo ok`，失败自动清空当前设备，AdbKitApp 监听 currentDevice 并自动回到首页
- ~~未连接设备时首页空状态无引导~~ → 未连接设备时首页展示欢迎卡片与连接步骤，并可直接唤起扫描
- ~~首次启动直接申请 MANAGE_EXTERNAL_STORAGE，无解释说明~~ → 首次启动或授权缺失时先弹窗说明用途，再由用户确认跳转系统设置
- ~~连接失败提示不友好~~ → 解析 adb connect 输出并分类为 refused/unreachable/offline/auth/invalid，给出可操作的引导文案
- ~~首页缺少本机 IP 提示~~ → IP 输入框下方展示本机 IP，辅助输入同网段地址
- ~~缺少 USB 调试授权引导~~ → 已实现：首页/远程控制/Fastboot 等页面在未授权时提供 USB 调试授权引导与重试入口

### 2. 设备信息（完成度 8/10）

**优点**
- 信息维度全面，分组清晰，支持复制

**不足**
- 部分字段（如 total_storage/available_storage）使用 `df` 解析，格式不稳定
- ~~缺少 GPU、屏幕刷新率、相机、传感器、电池健康度等进阶信息~~ → 已增加 GPU、屏幕刷新率、电池健康度、相机数量等字段
- ~~不支持导出/分享为文本/JSON~~ → 已支持通过系统分享一键发送设备信息文本；已支持导出 JSON 格式
- ~~无设备对比/历史记录~~ → 已实现：刷新时自动保存电池/存储/温度快照，历史弹窗以 Canvas 折线图展示电池与存储占用趋势；并支持 JSON 导出分享

### 3. 实用工具（完成度 6/10）

**优点**
- 工具数量多，覆盖截图、录屏、重启、输入、系统控制等

**不足**
- ~~录屏无法中途停止，固定 180 秒；无实时预览~~ → 已实现中途停止（Snackbar 提供停止按钮，取消后通过 pkill 发送 SIGINT 停止设备端 screenrecord 进程）
- ~~截图/录屏结果未在应用内展示，也无保存到本地提示~~ → 已实现（截图/录屏完成或停止后在 Snackbar 显示设备侧路径 /sdcard/screenshot_adbkit.png 与 /sdcard/screenrecord_adbkit.mp4）
- ~~亮度只能设置绝对值，未读取当前值~~ → 已实现（打开亮度对话框时读取当前值作为滑块默认值）
- ~~WiFi/蓝牙/飞行模式没有状态回显，容易误操作~~ → 已实现（先读取当前状态，切换后回显目标状态 ON/OFF）
- ~~系统属性只能查看，无法修改（缺少 setprop UI）~~ → 已实现：实用工具页有「系统属性」入口，可查看全部 getprop 并在弹窗内输入属性名/值执行 setprop
- ~~缺少批量脚本/宏录制能力~~ → 已支持命令收藏夹（20 条，持久化）与一键执行收藏夹脚本；支持将收藏夹保存为命名宏、独立运行、删除；支持将输出结果保存/分享

### 4. 远程控制（完成度 7.0/10）

**优点**
- 自研 scrcpy 兼容协议，H.264 硬解，支持触控/手势/虚拟按键
- 支持熄屏控制、仅浏览模式
- 连接前检测当前设备，未连接时给出引导提示

**不足**
- ~~连接入口缺少设备检测/引导，用户不知如何启动~~ → 已增加未连接设备提示
- 音频开关有 UI 但协议层 audio 路径未完整闭环 → 已在 UI 下方添加说明：音频转发需设备端支持，当前预留开关
- ~~熄屏方案依赖 `SurfaceFlinger` 反射/service call，兼容性差~~ → 已实现多命令 fallback：service call SurfaceFlinger、亮度调节、svc power goToSleep/input keyevent 依次尝试，并回显状态
- ~~缺少键盘输入、剪贴板同步、文件拖放等高级能力~~ → 已支持投屏界面文本输入，并可将主机剪贴板内容粘贴为设备输入（通过 input text 注入）
- ~~无码率/帧率自适应，弱网体验差~~ → 已增加 FPS 低帧率检测与弱网模式提示，可一键应用低分辨率/低码率预设
- 不支持多设备同时投屏

### 5. 文件管理（完成度 6.5/10）

**优点**
- 基础浏览、上传、下载、删除、新建文件夹都有
- 删除操作新增二次确认（可关闭），降低误删风险
- 上传/下载状态栏显示文件名与大小

**不足**
- ~~不支持文件重命名、移动、复制、批量选择~~ → 已支持重命名、移动、复制；批量选择与批量删除（长按进入多选，顶部工具栏提供全选/批量删除/取消）
- ~~文件图标未按类型区分~~ → 已实现（目录/APK/图片/视频/音频/文档/代码/压缩包等均有不同图标与颜色）
- ~~大文件 pull/push 仍无字节级进度条（仅文件名/大小 + transferring 文本）~~ → 已实现：AdbService 增加 pullFileWithProgress / pushFileWithProgress，按文件大小轮询显示 LinearProgressIndicator
- ~~缺少文本预览、图片预览、APK 安装快捷入口~~ → 已支持文本/图片在线预览，APK 文件下拉菜单新增「安装」入口，点击直接调用 pm install
- ~~根目录/系统目录访问依赖 root 权限，但无 root 检测与提示~~ → 已进入 /、/data、/system、/vendor、/product、/sbin 等受保护目录且无 root 时，页面顶部显示红色警告提示

### 6. 应用管理（完成度 7.5/10）

**优点**
- 用户/系统应用分类、搜索、启动、停止、卸载、冻结、备份完整
- 强制停止、清除数据、卸载、禁用均增加二次确认弹窗

**不足**
- ~~卸载/清除数据/冻结等危险操作无二次确认~~ → 已实现
- ~~应用详情加载较慢，缺少图标、组件信息~~ → 应用详情弹窗已显示应用图标、版本/路径/SDK 等详情、权限列表、活动/服务/接收器/内容提供器数量
- ~~未提供「导出应用列表」~~ → 已支持导出当前分类的应用列表到 /sdcard/app_list_adbkit.txt
- ~~缺少「批量备份」~~ → 已支持批量备份当前分类/过滤列表的 APK 到 /sdcard/app_backup_adbkit
- ~~安装 APK 后不会清理缓存临时文件~~ → 安装流程完成后自动删除本地 content picker 缓存与设备侧 /data/local/tmp/install_temp.apk

### 7. 进程管理（完成度 6/10）

**优点**
- 运行应用/所有进程切换、内存概览、搜索、结束进程都有

**不足**
- ~~CPU 占用数据未实时刷新~~ → 已支持每 5 秒自动刷新进程/运行应用/内存/CPU 数据
- ~~无进程排序（按内存/CPU/名称）~~ → 已支持按内存、PID、进程名、CPU 排序，并支持升/降序切换
- ~~结束进程后不会自动刷新列表~~ → killProcess / forceStopApp 成功后均会调用 refresh()
- ~~缺少进程详情（命令行、线程数、启动时间）~~ → 点击运行应用/所有进程项的「详情」图标可查看命令行、线程数、父进程、CPU 时间、常驻页

### 8. 命令终端（完成度 7.5/10）

**优点**
- ADB Shell / ADB 命令双模式、历史记录、快捷命令、彩色输出
- 命令历史已持久化到 DataStore（最多 50 条，可关闭）

**不足**
- ~~命令历史未持久化（重启丢失）~~ → 已实现
- ~~输出为纯文本，长输出无搜索/复制~~ → 已支持搜索过滤输出与一键复制全部输出
- ~~不支持命令收藏、脚本保存~~ → 已支持命令收藏夹（DataStore 持久化，最多 20 条）
- ~~交互式命令（如 `top`）会阻塞，缺少 cancel 能力~~ → 已实现（执行中的命令可取消，底层进程会被终止）

### 9. Fastboot（完成度 6.5/10）

**优点**
- 刷机、重启、解锁/锁定、擦除、自定义命令已具备
- 镜像文件选择器已实现，可选择本地镜像文件
- 刷机、解锁 BL、锁定 BL、擦除分区均增加二次确认

**不足**
- ~~镜像路径输入框旁的文件选择按钮未实现~~ → 已实现
- ~~Fastboot 在手机端运行通常需要 root/特殊权限，缺少权限检测与引导~~ → 已实现：检测 root 与 USB 权限，无设备/无权限时给出对应引导文案
- ~~无刷机包验证、无备份分区能力~~ → 已支持镜像 MD5 校验与期望 MD5 比对；已支持列出 /dev/block/bootdevice/by-name 分区并使用 dd 备份到 /sdcard，支持从设备侧镜像还原分区
- ~~刷机过程无进度、无日志保存、无失败恢复建议~~ → 已支持将刷机/命令输出保存到下载目录

### 10. 设置与全局（完成度 7/10）

**优点**
- ADB 路径配置、自动检测、诊断信息完整
- 深色模式、动态取色、语言切换已有基础

**不足**
- ~~语言切换目前仅字符串资源抽象，未与系统语言/应用内切换完全绑定~~ → 已实现：语言设置支持「跟随系统/中文/英文」，AdbKitApp 根据系统 Locale 自动选择中文或英文
- ~~缺少帮助文档、FAQ、反馈入口~~ → 已实现：新增 HelpScreen，从设置页和首页菜单可进入，提供 USB 调试、无线配对、权限、连接错误、Fastboot 等 FAQ
- ~~缺少崩溃上报、使用统计~~ → 已实现基础崩溃日志：CrashHandler 保存崩溃堆栈并追加 App 版本/设备/系统信息，设置页可分享/清空崩溃日志，支持邮件反馈
- ~~未做新手引导/空状态引导~~ → 已实现：首次启动展示 OnboardingDialog，首页未连接设备展示欢迎卡片与操作步骤

## 三、核心问题与风险

1. ~~**连接历史/命令历史未持久化**~~ → 已修复（DataStore 持久化）
2. ~~**危险操作缺少确认**~~ → 已修复（应用/文件/Fastboot 危险操作均增加二次确认弹窗）
3. ~~**Fastboot 文件选择器未实现**~~ → 已修复（镜像路径支持文件选择）
4. **屏幕串流兼容性风险**：反射/SurfaceFlinger/service call 在不同厂商 ROM 上差异大
5. ~~**错误处理薄弱**~~ → 部分修复（统一空设备占位页、动态参数已 shell 转义）
6. ~~**无 root 检测与引导**~~ → 部分修复（文件管理器访问 /data、/system 等目录时检测并提示 root）
7. ~~**权限管理粗糙**：首次启动直接申请 `MANAGE_EXTERNAL_STORAGE`，无解释说明~~ → 已修复：首次启动或授权缺失时先弹窗说明用途，再由用户确认跳转系统设置
8. ~~**无单元测试/自动化测试**~~ → 已新增 JUnit + Kotlinx-coroutines-test 基础设施，覆盖 shellQuote、CommandResult、classifyConnectionError、mapBatteryHealth、verifyImageMd5 等核心纯函数
9. ~~**国际化不完整**~~ → 中英双语已实现，语言设置支持「跟随系统」，缺失字符串已补齐
10. ~~**文档缺失**~~ → 已新增 docs/USER_GUIDE.md 用户手册，README 已链接用户手册与路线图；仍缺少 API 文档、贡献指南
